# Contributing to the Trail Running Glossary

Thank you for your interest in contributing! We want to make this resource the best it can be for the global community.

## Quick Start
1.  **Search** the `glossary/` folder to ensure your term doesn't already exist.
2.  **Use the Template** below to format your contribution.
3.  **Submit** a Pull Request (PR) with your changes.

## Copy-Paste Template
When creating a new file, copy this entire block and fill in the details:

```markdown
# [English Term]

## Meta
- **Definition:** [The canonical definition/explanation of the term.]
- **Category:** [e.g., ITRA System, Gear, Medical, Environment]
- **Tags:** [tag1, tag2]
- **Reference URL:** [Link to official doc] (sample)
- **Reference Sentence:** [Contextual usage sentence] (sample)

## English (EN)
- **Term:** [Term in English]
- **Status:** Draft

## French (FR)
- **Term:** [Term in French]
- **Status:** Draft

## Spanish (ES)
- **Term:** [Term in Spanish]
- **Status:** Draft

## Chinese (ZH)
- **Term:** [Term in Chinese]
- **Status:** Draft
- **Notes:** [Optional specific context]
```

## Detailed Workflow (GitHub Web UI)

### Adding a New Term
1.  **Navigate:** Go to the `glossary/` folder in this repository.
2.  **Create:** Click **Add file** > **Create new file**.
3.  **Name:** Name your file using the English term in lowercase with dashes (e.g., `performance-index.md` or `trail-running-shoes.md`).
4.  **Paste:** Paste the template code from above into the text editor.
5.  **Edit:** Replace the bracketed text `[...]` with your content.
6.  **Commit:**
    *   Scroll down to "Commit changes".
    *   Write a short description (e.g., "Add Trail Running Shoes").
    *   **Crucial:** Select **"Create a new branch for this commit and start a pull request"**.
    *   Click **Propose changes**.

### Editing an Existing Term
1.  **Navigate:** Open the file you want to change in the `glossary/` folder.
2.  **Edit:** Click the **Pencil icon** (Edit this file) in the top right of the file view.
3.  **Update:** Make your corrections or add a new language translation.
4.  **Commit:**
    *   Scroll down to "Commit changes".
    *   Write a short description (e.g., "Fix typo in Chinese definition").
    *   Select **"Create a new branch for this commit and start a pull request"**.
    *   Click **Propose changes**.

## Review Process
Once you submit a Pull Request:
1.  **Automated Check:** A bot will check if your file follows the correct format. Look for a green checkmark or a red "X" at the bottom of your PR page.
2.  **Editorial Review:** A project maintainer will review the content for accuracy.
3.  **Merge:** Once approved, your contribution will be merged and live!
